# Excel → Markdown 仕様書生成スキル

## 前提条件

```bash
pip3 install openpyxl oletools
```

スキル実行時にインストール済みか確認し、未インストールなら自動インストールする。

## ディレクトリ規約

このスキルはプロジェクトルート内の以下の構成を前提とする:

```
<project_root>/
├── xlms/    ← Excel ファイル（.xlsx/.xlsm）の格納場所
└── docs/    ← 生成した MD 仕様書の出力先
```

- 対象ファイルが `xlms/` にない場合は `xlms/` にコピーしてから処理する
- MD 仕様書は常に `docs/<元ファイル名（拡張子除く）>_仕様書.md` に出力する

## 実行手順

### Step 1: ファイルとプロジェクトルートを特定

1. 対象 Excel ファイルのパスを確認する
2. プロジェクトルートを特定する（`xlms/` と `docs/` が存在するディレクトリ）
3. ファイルが `xlms/` 以外にある場合は `xlms/` にコピーする
4. 拡張子で処理を分岐:
   - `.xlsx` → VBA抽出ステップをスキップ
   - `.xlsm` / `.xlsb` → 全ステップ実行

### Step 2: 解析スクリプト実行

```bash
python3 .cursor/skills/excel-to-md/scripts/extract_excel.py "<xlms/ファイルパス>"
```

スクリプトが以下のJSON構造を標準出力する:
- `sheets`: シート一覧（名前・行数・列数）
- `cells`: 全セルの値（data_only=True）
- `formulas`: 全数式（data_only=False）
- `merged_cells`: マージセル範囲
- `defined_names`: 名前付き範囲
- `buttons`: ボタン・マクロ割り当て
- `external_links`: 外部ファイル参照

### Step 3: VBA 抽出（.xlsm のみ）

```bash
olevba "<xlms/ファイルパス>" 2>&1
```

出力からモジュール単位でVBAコードを分離・整理する。

### Step 4: MD仕様書の構成

[reference.md](reference.md) のテンプレートに従い、以下のセクションで構成する:

1. **ファイル概要**: 種別・用途・構成一覧
2. **シート詳細**: 各シートのレイアウト・ヘッダー・データ領域
3. **名前付き範囲一覧**: 全名前付き範囲を表形式で
4. **数式一覧**: シートごとに全数式と説明
5. **ボタン・マクロ対応**: ボタンラベル・割り当てマクロ
6. **VBAモジュール仕様**: 各モジュールの処理フロー解説
7. **DB接続・外部連携**: DSN・テーブル・SQL文（該当する場合）
8. **データフロー**: 処理の流れを図解（テキストベース）
9. **セキュリティ注意**: olevba警告の記載

### Step 5: 出力

```
docs/<元ファイル名（拡張子除く）>_仕様書.md
```

に生成した MD を保存する。`docs/` が存在しない場合は作成する。

## スコープ外

以下は完全な再現が困難なため、明示的にスコープ外とする:
- セル書式（色・罫線・フォントサイズ）
- 条件付き書式
- グラフ・画像
- 印刷設定

必要な場合はスクリーンショットで補完する旨を記載する。

## 判断基準

- セルに値がない行は省略してよい
- VBAコードは処理フローの日本語解説を優先し、コード全文は補足として掲載
- SQL文が含まれる場合はテーブル一覧・カラム説明を別セクションで整理
- 数式に科学的・業務的意味がある場合（半減期計算など）はコメントで補足
