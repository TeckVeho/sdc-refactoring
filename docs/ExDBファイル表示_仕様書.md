# ExDBファイル表示 仕様書

> **ファイル種別**: .xlsm（マクロ付き）
> **用途**: Oracle DB の任意のテーブルをユーザーが選択・条件設定してデータを抽出・表示する汎用DBブラウザツール。テーブル定義は Access DB（`テーブル一覧.accdb`）で管理する
> **VBA プロジェクトサイズ**: 約11.3MB（抽出結果シートに最大30,000行×140列のデータを保持）
> **外部連携**:
> - Oracle DB（ODBC `DSN=ricdb`）：各業務テーブルのデータ抽出
> - Access DB（`\\163.59.144.156\ExRicSys\テーブル一覧.accdb`）：テーブル定義・項目情報の管理

## 2. シート詳細

### 2.1 使い方

**目的**: このツールの操作手順（Ⅰ:データ抽出方法、Ⅱ:テーブル設定登録・修正方法）を記載した説明シート。

#### 操作概要（セル記載内容）

**Ⅰ データ抽出方法**:
1. 「項目TB」シートでテーブルを選択し「選択」コマンドをクリック → 「抽出」シートに遷移
2. 「抽出」をクリック
3. 表示したい項目は項目名または表示列を右クリックで選択（「する」に切替）
4. 表示不要の場合はC列の表示を「しない」にする
5. 必要に応じて抽出条件を設定（文字数は桁数と一致させること）
6. 最大抽出数以上データがある場合は条件を設定して再度抽出（最大30,000件）
7. 抽出結果を新しいBookに記録する場合は「Bookに記録」を使用

**Ⅱ テーブル設定登録・修正方法**:
1. 「項目TB」のテーブル一覧で新規/修正行を選択
2. 「Table登録」シートで入力して「登録」をクリック
3. 必須: テーブル名、DBテーブル名、DB、関係装置、項目名称、識別名、属性、桁数

## 6. VBA モジュール仕様

### 6.1 ThisWorkbook

#### `Workbook_Open()`

**処理概要**: ブック起動時に各シートを保護し、画面をクリアして最大抽出件数を 30,000 に設定し、テーブル一覧を表示する。

```vba
Private Sub Workbook_Open()
    Worksheets("抽出").Select
    Call シート保護
    Call AllCls
    Range("MaxDataN") = 30000
    Call テーブル一覧表示
End Sub
```

#### `Workbook_BeforeClose()`

**処理概要**: 保存ダイアログを抑止してブックを閉じる。

---

### 6.2 テーブル選択

#### `テーブル指定()`

**処理概要**: 「項目TB」シートでアクティブなセルのテーブルを選択し、対応するテーブル名・DBテーブル名・DB・説明を「抽出」シートの対応セルに設定する。その後 `項目名表示()` を呼び出す。

**処理フロー**:
1. アクティブセルが `Table` 範囲内かチェック
2. `T_TableNo`・`ReadDB`・`DB_FileName`・`DBName`・`F_Setumei` に値を設定
3. `項目名表示()` → Access DB の `項目一覧` から項目を取得して `TyusyutuTB` に表示

#### `項目名表示()`

**処理概要**: Access DB の `項目一覧` テーブルから指定 `テーブルNo` に対応する項目を取得して「抽出」シートに書き込む。

```sql
SELECT 表示順, '', 項目名称, 説明, 識別子, 属性, 桁数
FROM 項目一覧
WHERE テーブルNo = <テーブルNo>
ORDER BY 表示順
```

---

### 6.3 抽出設定

#### `データ抽出()`

**処理概要**: 「抽出」シートの設定（表示項目・条件・ソート）に基づいてOracle DBに動的SQLを発行し、結果を「抽出結果」シートに表示する。

**処理フロー**:
1. 入力検証（項目選択あり・ソート重複なし・DBName 設定あり）
2. Oracle DB接続文字列を設定: `DSN=ricdb;UID=<DBName>;PWD=t6101`
3. `TyusyutuTB`（項目一覧）から抽出条件（H/I列）を読み取り WHERE 句を生成
   - DATE型の場合: `TO_DATE(...,'yyyy/mm/dd hh24:mi:ss')`
   - その他: 文字列比較
4. 表示が「する」の項目の識別子を SELECT 句に追加
5. ソート設定に応じて `ROW_NUMBER() over(ORDER BY ...)` または `ROWNUM` を追加
6. `Disp_Sheet()` でデータを抽出結果シートの12行目から書き込み
7. DATE型カラムに `yyyy/mm/dd hh:mm` 書式を適用
8. 30,000件以上の場合は警告メッセージ
9. 「新しいBookに記録しますか？」の確認

**主要 SQL**:
```sql
-- ソートキーあり
SELECT ROW_NUMBER() over(ORDER BY <識別子1>, <識別子2>) as row_number,
       <識別子1>, <識別子2>, ...
FROM <DBテーブル名>
WHERE <識別子> >= '<下限値>' AND <識別子> <= '<上限値>' AND
      ROWNUM <= 30000

-- ソートキーなし
SELECT ROWNUM, <識別子1>, <識別子2>, ...
FROM <DBテーブル名>
WHERE ROWNUM <= 30000
```

#### `新しいBookに記録()`

**処理概要**: 「抽出結果」シートを新しいブックにコピーしてユーザーに提示する。

#### `全て表示_非表示()`

**処理概要**: ボタンテキストを「全て表示」↔「全て非表示」で切替えながら、`TyusyutuTB` の全行の表示列（B列）を「する」/空 に一括設定する。

### 6.4 テーブル一覧

#### `テーブル一覧表示()`

**処理概要**: 起動時に Access DB の `テーブル一覧` から全テーブルを取得して「項目TB」シートの `Table` 領域に表示する。

```sql
SELECT テーブルNo, 関連装置, テーブル名, DBテーブル名, 説明, DB
FROM テーブル一覧
ORDER BY 関連装置, DBテーブル名
```

#### `テーブル一覧修正登録()`

**処理概要**: 「項目TB」シートでアクティブ行のテーブルを「Table登録」シートに読み込む。新規の場合は最大テーブルNo+1 を設定する。

#### `テーブル登録()`

**処理概要**: 「Table登録」シートの入力内容を Access DB の `テーブル一覧` および `項目一覧` テーブルに INSERT/UPDATE する。変更有無（`HenKou1`/`HenKou2`）に応じて処理を分岐する。

```sql
-- テーブル一覧への登録
INSERT/UPDATE テーブル一覧 SET (テーブルNo, テーブル名, DBテーブル名, DB, 関連装置, 説明)

-- 項目一覧の既存データ削除後に再登録
DELETE FROM 項目一覧 WHERE テーブルNo = <テーブルNo>
INSERT INTO 項目一覧 (テーブルNo, 表示順, 項目名称, 識別子, キー, 属性, 桁数, 説明)
```

**処理概要**: 抽出結果シートのデータ・抽出設定・Table登録シートを全クリアし、初期状態に戻す。

---

### 6.9 ユーティリティ

#### `一覧登録()`

**処理概要**: 「項目TB」シートのテーブル一覧を Access DB の `テーブル一覧` テーブルに一括登録する（初期データ投入用）。

#### `シートデータコピー()` (Ctrl+b)

**処理概要**: アクティブシートのデータを「Table登録」シートにコピーする（運用開始時の一括登録用）。通常は `Exit Sub` で無効化されている。

## 7. DB 接続・外部連携

### Oracle DB（ODBC）

| DSN 名 | UID | PWD | 用途 |
|---|---|---|---|
| `ricdb` | `<DBName セルの値>` | `t6101` | 各業務テーブルのデータ抽出・参照 |

> **UID の動的設定**: 接続文字列は `"DSN=ricdb;UID=" & Range("DBName") & ";PWD=t6101"` で生成される。`DBName` セルにユーザーが入力した DB 名が UID として使用される。

### Access DB（OLE DB）

| 接続先パス | プロバイダー | 用途 |
|---|---|---|
| `\\163.59.144.156\ExRicSys\テーブル一覧.accdb` | Microsoft.Ace.OLEDB.12.0 | テーブル定義・項目一覧の管理 |

### Access DB テーブル一覧

| テーブル名 | 主な用途 | 主要カラム |
|---|---|---|
| `テーブル一覧` | 抽出可能なDBテーブルのメタ情報 | `テーブルNo`, `テーブル名`, `DBテーブル名`, `DB`, `関連装置`, `説明` |
| `項目一覧` | 各テーブルのカラム定義 | `テーブルNo`, `表示順`, `項目名称`, `識別子`, `キー`, `属性`, `桁数`, `説明` |

### 抽出可能な Oracle DB テーブル

Oracle DB 上の業務テーブル（`zaiko`, `sehmst`, `sengnr1` 等）から `テーブル一覧.accdb` に登録されているものすべてが抽出可能。テーブルの一覧は Access DB によって管理されるため、このファイルには含まれない。

## 8. データフロー

```
【起動フロー】
  Workbook_Open()
        ↓
  シート保護() → 項目TB/抽出/Table登録 を保護
  AllCls() → 全シートをクリア
  MaxDataN = 30000 に設定
  テーブル一覧表示():
    Access DB(テーブル一覧.accdb): SELECT FROM テーブル一覧
        → 項目TBシートの Table 領域に書込

【テーブル選択・項目表示フロー】
  項目TBシート: テーブル行を選択
        ↓ [選択]ボタン → テーブル指定()
  T_TableNo / ReadDB / DB_FileName / DBName / F_Setumei を設定
  項目名表示():
    Access DB(テーブル一覧.accdb): SELECT FROM 項目一覧 WHERE テーブルNo=<N>
        → 抽出シートの TyusyutuTB に書込

【データ抽出フロー】
  抽出シート:
    - 表示したい項目の B 列を右クリックで「する」に設定
    - H/I 列に抽出条件（下限/上限）を入力
    - J 列にソート設定（ｿｰﾄ1/ｿｰﾄ2）
        ↓ [抽出]ボタン → データ抽出()
  WHERE 句・SELECT 句・ORDER BY 句を動的生成
  Oracle DB(DSN=ricdb, UID=<DBName>): データ取得
        → 抽出結果シートの12行目〜に表示
  DATE型カラムに書式設定
  30000件超え→警告
  「新しいBookに記録しますか？」→ 新しいBookに記録()

【テーブル定義登録・修正フロー】
  項目TBシート: テーブル行を選択
        ↓ [テーブル設定の登録・修正]ボタン → テーブル一覧修正登録()
  新規: テーブルNoを最大値+1で設定 → Table登録シートへ
  修正: Access DB から既存定義を読込 → Table登録シートに表示
        ↓ Table登録シートに入力後 [登録]ボタン → テーブル登録()
  入力チェック → Access DB テーブル一覧/項目一覧 に INSERT/UPDATE
        → テーブル一覧表示() で項目TBシートを更新
```

## 9. セキュリティ注意事項

| 種別 | キーワード | 内容 |
|---|---|---|
| AutoExec | `Workbook_Open` | 起動時に保護設定・クリア・テーブル一覧表示が自動実行される |
| AutoExec | `Workbook_BeforeClose` | 保存ダイアログが自動で抑止される |
| Suspicious | `Open` | ファイルを開く操作が含まれる（Access DB への接続） |
| Suspicious | `Command` | ADO の `ADODB.Command` オブジェクトを使用する（PowerShell ではない） |
| Suspicious | `Call` | モジュール間の内部呼び出し |
| Suspicious | `Windows` | ウィンドウ操作（複数ブック操作時の正常な動作） |
| Suspicious | `Chr` | Chr(13)（改行）を使用したメッセージ生成 |
| Suspicious | Hex/Base64 Strings | xlsm 構造上の正常なバイナリデータ |
| **IOC** | `163.59.144.156` | **VBAコードおよびAccess DBパスにDBサーバーIPがハードコーディング** |

> **重大な注意点**:
> 1. Access DB の接続先パス（`\\163.59.144.156\ExRicSys\テーブル一覧.accdb`）がVBAコードにハードコーディングされており、IPアドレスが平文で埋め込まれています。
> 2. Oracle DB の ODBC 接続では、パスワード `t6101` が平文でハードコーディングされています。
> 3. Oracle DB の UID は `DBName` セルの値を動的に使用するため、ユーザーが任意の UID でDBに接続できます。
> 4. このファイルのサイズが11.3MBと大きいのは、「抽出結果」シートに最大30,000行×140列のデータが格納されているためです。機密データが含まれている可能性があります。
