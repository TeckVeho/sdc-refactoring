# ExJMM90φ記入用紙 仕様書

> **ファイル種別**: .xltm（マクロ付きテンプレート）
> **用途**: 1号機照射の JMM 90φPE 丸棒線量測定記録用テンプレート。DB から受付情報を読み込み、測定値入力→線量計算→報告書印刷→ファイル保存を行う。
> **VBA プロジェクトサイズ**: 103 KB
> **外部連携ファイル**: `D:\JMM線量\<年度>\<受付番号>JMM90.xlsm`（保存先）
> **備考**: 元ファイルは `.xltm`（Excel マクロ有効テンプレート）。`ExJMM60φ記入用紙.xltm` の 90φ 版であり、VBA 構造は同一。シートサイズ・名前付き範囲の配置・最大行数が異なる。

## 5. ボタン・マクロ対応

60φ版では「ｶｰｿﾙ右」「保存」「終了」の 3 ボタンがあったが、**90φ版にはボタンなし**。

測定値シートの `ｶｰｿﾙ右`（`CarsolRight`）・`保存`（`フォルダー作成と保存`）・`終了`（`Bookを閉じる`）の各マクロは VBA モジュールに定義されているが、シート上からボタン経由では呼び出されない構成。

## 6. VBA モジュール仕様

VBA の全モジュール・プロシージャは `ExJMM60φ記入用紙_仕様書.md` の §6 と同一。以下に差分のみ記載する。

### 差分: 保存先ファイル名

`フォルダー作成と保存` 内の `SaveAs` で生成されるファイル名が `JMM60.xlsm` ではなく `JMM90.xlsm` になる（VBA コード内の文字列参照先が異なる）。

### 差分: 印刷範囲

`印刷範囲の設定` の計算式は同一だが、90φ の 1 バッチあたりの本数・行数が異なるため、実際に設定される行範囲が異なる。

---

詳細な VBA 仕様は [`ExJMM60φ記入用紙_仕様書.md`](ExJMM60φ記入用紙_仕様書.md) の §6 を参照のこと。

## 7. DB 接続・外部連携

60φ版と同一。

### ODBC 接続設定

| DSN 名 | UID | PWD | 用途 |
|---|---|---|---|
| `ricdb` | `ric` | `t6101` | メイン実績 DB（Oracle） |
| `ricdb` | `rich` | `t6101` | 代替接続（コメントアウト） |

### 参照テーブル一覧

| テーブル名 | 主な用途 | 主要カラム |
|---|---|---|
| `keicode` | 計算式コードマスタ | `keisask`, `yflg1〜5`, `consta1〜5`, ..., `temper1〜3` |
| `senkind` | 線量計種類マスタ | `sensyu` |
| `zaiko` | 在庫テーブル（受付情報） | `uno`, `kaisyacd`, `nyukasu`, `siteisn`, `kagensn`, `jyougsn`, `nyukabi`, `syouso` |

### 主要 SQL 文

```sql
-- 有効な計算式コードを取得
SELECT keisask FROM keicode WHERE yflg1='1' AND SUBSTR(keisask,2,1)='1' ORDER BY keisask

-- 計算式パラメータ（温度 1 の係数）
SELECT TO_NUMBER(conste1), TO_NUMBER(constd1), TO_NUMBER(constc1),
       TO_NUMBER(constb1), TO_NUMBER(consta1), TO_NUMBER(temper1), TO_NUMBER(yflg1)
FROM keicode WHERE keisask='<計算式コード>'

-- 線量計種類マスタ
SELECT sensyu FROM senkind

-- 在庫データ取得（スタートフォーム）
SELECT nyukasu, siteisn, kagensn, jyougsn, nyukabi
FROM zaiko WHERE uno='<受付番号>' AND syouso='1' AND kaisyacd='<会社コード>'
```

### 外部ファイル連携

| ファイル名パターン | 説明 |
|---|---|
| `D:\JMM線量\<年度>年度\<受付番号>JMM90.xlsm` | 測定記録の保存先（90φ 専用） |

## 8. データフロー

60φ版と同一のフロー（起動→StartForm→受付番号入力→測定値入力→線量計算→保存）。

```
【起動フロー】
  ブックを開く（Workbook_Open）
       ↓ 初期化
  DB(senkind): 線量計種類取得 → SenKindTB へ書き込み（R2:R4）
       ↓
  DB(keicode): 有効計算式コード取得 → KeiCode へ書き込み（T2:T3）
       ↓
  StartForm 表示（受付番号入力）
       ↓ DB(zaiko): 受付番号で在庫データ取得
  既存ファイルあり → Workbooks.Open
  新規             → 入力欄をセット

【測定値入力フロー】
  受付本数（D3）変更 → 印刷範囲を自動設定
  計算式コード（H3）変更 → DB(keicode) からパラメータ取得 → SikiPara（Y2:AE6）
  ABS 値（H列）入力 → 線量計算 → I 列に測定線量を書き込み

【保存フロー】
  フォルダー作成と保存 → D:\JMM線量\<年度>\<受付番号>JMM90.xlsm に保存
```

## 9. セキュリティ注意事項

60φ版と同一の olevba 検知結果。

| 種別 | キーワード | 内容 |
|---|---|---|
| AutoExec | `Workbook_Open` | 起動時に自動で `初期化` が実行される |
| AutoExec | `Workbook_BeforeClose` | 閉じる前に保存確認が実行される |
| AutoExec | `CommandButton1_Click` | UserForm のボタンで DB クエリとファイル操作が実行される |
| AutoExec | `Worksheet_Change` | セル変更ごとに DB クエリ・計算が自動実行される |
| Suspicious | `Open` / `Write` / `Output` | ファイル操作系ステートメントを使用 |
| Suspicious | `MkDir` | ローカルディスクにディレクトリを作成する |
| Suspicious | `Chr` | ダイアログの改行に使用（難読化ではなく通常用途） |
| Suspicious | `Hex Strings` / `Base64 Strings` | フォームの UI 設定バイナリに由来する可能性が高い |
| 認証情報 | `DSN=ricdb;UID=ric;PWD=t6101` | Oracle DB の接続文字列がコード内にハードコードされている。**パスワードの管理に注意** |
