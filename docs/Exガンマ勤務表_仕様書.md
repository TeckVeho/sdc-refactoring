## 6. VBA モジュール仕様

### 6.1 ThisWorkbook.cls

#### `Workbook_Open()`

**処理概要**: ブック起動時に `スタート` を呼び出し資格確認・初期化を行う。

#### `Workbook_BeforeClose(Cancel As Boolean)`

**処理概要**: 未記録の場合、保存確認ダイアログを表示する。

**処理フロー**:
1. デバッグモードなら終了確認のみ
2. 未記録 かつ 作成・本人資格の場合、「計画の保存をしますか」を表示
3. はい → `勤務記録` を実行してから終了
4. キャンセル → 終了中断
5. セル移動設定を元に戻し、保存済みフラグをセット

#### `Workbook_BeforeSave()`

**処理概要**: 通常の保存を禁止（DebugF=`*` の時のみ保存可）。

#### `Workbook_BeforePrint()`

**処理概要**: 各シートに応じた印刷設定を自動適用する。勤務計画シートは「勤務計画印刷用」での印刷を推奨する確認ダイアログを表示する。

---

### 6.2 Sheet1.cls（勤務計画シート）

#### `Worksheet_Change()`

**処理概要**: `S_Day`（期間始め日付）が変更されたとき、日付バリデーション・画面体裁更新・勤務時間テーブル読込・勤務記号セットを自動実行する。

**処理フロー**:
1. 入力日付の日が11日かチェック（11日でなければエラー）
2. `表体裁` → 画面体裁更新
3. `勤務時間テーブル読込` → DBから勤務時間テーブルを取得
4. `勤務記号` → 勤務記号ヘッダー行を設定

#### `Worksheet_SelectionChange()` / `myActivCell()`

**処理概要**: セル選択時にコピー元行から勤務記号をコピー先セルへ貼り付ける操作を実現。

**動作**:
- KinmuMoto行（行8）のセルを選択 → コピーモードに
- Kinnmu1エリアのセルを選択 → `PasteSpecial(xlPasteValues)` で貼り付け
- 作成無資格者（`mpSikaku=0`）は自分の行、休み希望・有休等の記号のみ貼り付け可能

#### `Worksheet_BeforeRightClick()`

**処理概要**: Kinnmu1エリアの右クリックで勤務記号を消去する。資格に応じて消去できる記号を制限。

---

### 6.3 勤務データ記録.bas

#### `記録Main()`

**処理概要**: 資格確認後に `勤務記録` を呼び出す。

#### `勤務記録()`

**処理概要**: 作成資格に応じて勤務計画・トイレ掃除データをAccess DBに保存する。

**処理フロー**:
1. 日付確認
2. 作成資格者（`mpSikaku=1`）：
   - `勤務時間テーブル記録` → 勤務時間テーブル更新
   - `記録("トイレ")` → トイレ掃除担当を `勤務予定表` テーブルへINSERT/UPDATE
   - `記録("ガンマ")` → ガンマ勤務計画を `勤務予定表` テーブルへINSERT/UPDATE
3. 本人のみ資格（`mpSikaku=0`）：
   - `記録("トイレ")`（追加可能時のみ）
   - `記録("ガンマ")`（自分の行のみ）
4. 「記録しました」メッセージ表示

#### `記録(勤務)`

**処理概要**: 指定シートの勤務データを `勤務予定表` テーブルへINSERT/UPDATE。

フィールド構成（35列）: 職場、勤務期間始、社員番号、表示順、日1〜日31

---

### 6.4 SQL_AccessExe.bas

#### `Open_AccessDB()`

**処理概要**: Access DB（`ガンマ勤務計画DB.accdb`）へMicrosoft.Ace.OLEDB.12.0でADODB接続を開く。

```vba
Public Const mpDBName As String = "\\163.59.144.156\ExRicSys\ガンマ勤務計画DB.accdb"
db.Provider = "Microsoft.Ace.OLEDB.12.0"
db.Open mpDBName
```

主要メソッド:
- `SQL_Access_Exe(mySql)` → SQL実行
- `SQL_Access_INSERT_UPDATE(myTBL, myKey, myD(), myN)` → INSERT/UPDATE汎用
- `SQL_Access_Delete(myTBL, myWhere)` → DELETE汎用
- `Disp_Access_Sheet(...)` → SQL結果をシートに貼り付け
- `Set_Access_Array(...)` → SQL結果を2次元配列に格納

## 8. データフロー

```
【起動フロー】
  ブック起動 (Workbook_Open)
       ↓
  スタート: Form資格でパスワード確認 → mpSikaku（資格レベル）設定
       ↓
  AccessDB ガンマ入出荷勤務 から社員一覧読込
       ↓
  AccessDB 労働時間表 から勤務時間テーブル読込
       ↓
  AccessDB 勤務予定表 から直近の勤務計画を読込 → 勤務計画シートに表示
       ↓
  画面表示完了

【日付入力フロー（期間始め変更時）】
  S_Day セルに年月日入力 (Worksheet_Change)
       ↓
  11日チェック（11日以外はエラー）
       ↓
  表体裁: 日付ヘッダー・曜日・色設定
       ↓
  勤務時間テーブル読込: AccessDB 労働時間表 から期間に対応するテーブル読込
       ↓
  勤務記号: KinmuMoto行に勤務記号をセット

【勤務計画入力フロー（勤務記号貼り付け）】
  KinmuMoto行（行8）のセルを選択 → コピーモード
       ↓
  Kinnmu1エリアの対象セルを選択
       ↓
  資格チェック（作成無資格者は本人行かつ指定記号のみ）
       ↓
  PasteSpecial(xlPasteValues) で勤務記号を貼り付け

【記録フロー】
  「記録」ボタンクリック
       ↓
  作成資格確認（mpSikaku）
       ↓
  勤務時間テーブル記録: AccessDB 労働時間表 を更新
       ↓
  記録("トイレ"): トイレ掃除 Toire1エリアデータ → AccessDB 勤務予定表 へINSERT/UPDATE
       ↓
  記録("ガンマ"): 勤務計画 Kinnmu1エリアデータ → AccessDB 勤務予定表 へINSERT/UPDATE
       ↓
  「記録しました」表示
```

## 9. セキュリティ注意事項

| 種別 | キーワード | 内容 |
|---|---|---|
| AutoExec | `Workbook_Open` | ブック起動時に自動実行。資格確認・DB接続が行われる |
| AutoExec | `Workbook_BeforeClose` | 終了時に自動実行。未記録チェック・自動保存が行われる |
| AutoExec | `ComCancel_Click` | フォームのキャンセルボタンが自動実行トリガーになる |
| AutoExec | `Worksheet_Change` | 日付入力時にDB接続が自動実行される |
| Suspicious | `Open` | ファイルオープン操作が含まれる可能性 |
| Suspicious | `Command` | PowerShellコマンドの実行の可能性 |
| Suspicious | `Call` | DLL呼び出しの可能性 |
| Suspicious | `Chr` | 文字列難読化の可能性（`Chr(13)` = 改行は正常用途） |
| Suspicious | Hex Strings | 16進数エンコード文字列が検出された |
| Suspicious | Base64 Strings | Base64エンコード文字列が検出された |
| IOC | `163.59.144.156` | ネットワーク上のIPアドレスがVBAコードに直書きされている |

> **重大な注意**:
> 1. Access DBのファイルパスがVBAコードに直書きされている：
>    ```
>    \\163.59.144.156\ExRicSys\ガンマ勤務計画DB.accdb
>    ```
> 2. Oracle DB接続パスワードが平文で存在する（SQL_Execution.bas）
> 3. 管理者パスワードが `使用方法!$C$53` セルに格納されている（シート保護なし時は参照可能）
