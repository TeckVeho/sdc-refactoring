# Exガンマ照射課実績集計 仕様書

> **ファイル種別**: .xltm（マクロ付きテンプレート）
> **用途**: ガンマ照射課の 1〜3 号機月次実績集計テンプレート。DB から照射実績・在庫数を取得し、標準ケース換算・入荷数合計を集計して報告書として出力する。
> **VBA プロジェクトサイズ**: 102 KB
> **外部連携ファイル**: なし（全データは DB から取得）
> **備考**: 元ファイルは `.xltm`（Excel マクロ有効テンプレート）。2 号機・3 号機の実績シートはテンプレート状態（セルが空）。`zaiko` 名前付き範囲は `#REF!` エラー状態。

### 2.2 1号機実績

**目的**: 1 号機の月次照射実績を明細表示する。DB から取得した照射データを行単位で表示し、標準ケース換算を計算する。テンプレート行（16〜17 行目）に数式が定義されており、`集計実行` 後にデータ行へコピーされる。

#### レイアウト構造

| セル範囲 | 名前付き範囲 | 内容 |
|---|---|---|
| `A1` | — | 年（`=Nenn & "年"`） |
| `B1` | — | 月（`=Tuki&"月"`） |
| `A2:C2` | — | 1 号機線源量（`=集計!D5`、マージ） |
| `D2:E2` | — | 「標準ケース換算合計」ラベル（マージ） |
| `F2` | — | 標準ケース換算合計（`=SUM(Q18:Q405)`） |
| `A3:B12` | — | 各種計算説明・係数ラベル（時間係数・箱係数の説明） |
| `J4` | — | 線量率（`=Rate` で集計シートから参照） |
| `J7` | — | 移動照射時の照射時間（`=10/J4`） |
| `I3:J7` | — | 標準ケースの条件パラメータ |
| `N3:O10` | `場所係数` | 場所名→場所係数のルックアップテーブル（8 行） |
| `S3:T11` | `照射位置` | 位置記号→照射位置名のルックアップテーブル（9 行） |
| `L15` | `Sunnpou` | 箱寸法記入フラグ（`=COUNTA(L18:L10000)`） |
| `M16:Q16` | `Siki` | テンプレート数式行（照射位置・場所係数・時間係数・箱係数・標準ケース換算） |
| `A18:L520` | `JissekiTB` | 実績データ本体（最大 502 行） |
| `A18:Q520` | `JissekiHyouji` | 実績データ表示範囲 |
| `A18:Q518` | `JissekiR1` | クリア対象範囲 |
| `A14:U342` | `jisseki` | 旧参照範囲（別名） |

#### テンプレート行数式（16 行目）

| セル | 数式 | 説明 |
|---|---|---|
| `M16` | `=IF(A16="","",VLOOKUP(TRIM(H16),照射位置,2,FALSE))` | 照射位置コード→照射位置名のルックアップ |
| `N16` | `=IF(M16="","",VLOOKUP(M16,場所係数,2,FALSE))` | 照射位置名→場所係数のルックアップ |
| `O16` | `=IF(A16="","",IF(M16="移動照射",(E16*L16+1360)/$E$5*(F16/10),I16/$J$7))` | 時間係数の計算（移動照射と固定照射で式が分岐） |
| `P16` | `=IF(N16="","",L16*E16/40)` | 箱係数（幅寸法 × 数量 / 40） |
| `Q16` | `=IF(N16="","",IF(M16="移動照射",O16/$J$7*20,N16*O16*P16))` | 標準ケース換算（移動照射と固定照射で式が分岐） |

### 6.2 データ抽出.bas

#### `集計実行()`

**処理概要**: 「集計実行」ボタンのエントリポイント。

**処理フロー**:
1. 確認ダイアログ（「1,2,3 号機の実績と入荷数を集計しますか？」）
2. `実績在庫抽出` を呼び出す（エラーがあればメッセージ表示して終了）
3. `入荷数合計` を呼び出す
4. 「集計終わりました」メッセージを表示

---

#### `入荷数合計()`

**処理概要**: 1〜3 号機それぞれの入荷数合計を DB から取得し、`Ric1IRI`〜`Ric3IRI` に書き込む。

**処理フロー**:
1. `Nenn` + `Tuki` から受付番号の範囲（`myRicS`〜`myRicE`）を生成
2. 各号機（syouso = 1, 2, 3）について：
   - `zaiko` テーブルから SUM(nyukasu) を取得
   - `rich.zaikor`（履歴テーブル）からも SUM(nyukasu) を取得し加算
   - 合計を `Ric1IRI` / `Ric2IRI` / `Ric3IRI` にセット

```sql
-- 在庫テーブル（当期）
SELECT SUM(nyukasu) FROM zaiko WHERE uno>'<年月>0000' AND uno<='<年月>9999' AND syouso='<号機>'

-- 履歴テーブル
SELECT SUM(nyukasu) FROM rich.zaikor WHERE uno>'<年月>0000' AND uno<='<年月>9999' AND syouso='<号機>'
```

---

#### `実績在庫抽出()`

**処理概要**: DB から 1〜3 号機の照射実績（受付番号・数量・指定線量・照射日等）を取得し、各号機実績シートに書き込む。

**処理フロー**:
1. 入力チェック（`Nenn`・`Tuki` の未入力確認）
2. 既に集計済みの場合は再集計確認
3. 箱寸法入力済みの場合は中止確認
4. 各号機（1〜3）のテーブル名を設定（`rich.SYOUJR1@RICDB` / `rich.SYOUJR2` 等）
5. 2 種類の接続文字列（`ricdb;UID=ric` / `ricdb;UID=rich`）でそれぞれ SELECT を実行
6. 結果をシートに書き込み

```sql
-- 1 号機照射実績（メイン DB）
SELECT sy.UNO, za.KAISYACD, za.SEHNCD, za.KAINAME, sy.SURYOU, sy.SITEISN,
       sy.KMMDD, sy.SYOITI, sy.SYOZI, sy.HANTENJ, '', ke.NAGASA
FROM rich.SYOUJR1@RICDB sy, rich.ZAIKOR@RICDB za, <keicode_table>

-- 2・3 号機照射実績
SELECT sj.uno, sj.suryou, sj.tuikaflg, sj.senkno, sj.senksyu, sj.atusa, sj.keisask,
       sj.kaisyacd, za.kainame, za.sehncd, sj.siteisn, sj.jyougsn, sj.kagensn,
       sj.sokutti, sj.sokutsn, sj.ondok, sj.kmmdd, sj.smmdd,
       sj.pass, sj.taitime, sj.sesti, sj.jituno*1
FROM rich.SYOUJR2 sj, rich.ZAIKOR za
```

### 6.3 箱寸法Ric1.bas

#### `箱寸法読込()`

**処理概要**: 1 号機実績シートのデータ行について、L 列（箱寸法）が空のものを DB の `SEHMST` テーブルから取得して補完する。

```sql
SELECT TRIM(NAGASA) FROM SEHMST WHERE KAISYACD='<会社コード>' AND SEHNCD='<製品コード>'
```

**処理フロー**:
1. 1 号機実績シートの 18 行目から A 列が空になるまでループ
2. L 列（箱寸法）が空の行のみ `SEHMST` を参照
3. 取得値が `*` / 空 / ≤0 の場合は空のまま、それ以外は数値として書き込む

### 主要 SQL 文

```sql
-- 在庫入荷数合計（当期）

-- 2・3 号機照射実績
SELECT sj.uno, sj.suryou, sj.tuikaflg, sj.senkno, sj.senksyu, sj.atusa,
       sj.keisask, sj.kaisyacd, za.kainame, za.sehncd, sj.siteisn, ...
FROM rich.SYOUJR2 sj, rich.ZAIKOR za
WHERE <条件>

-- 製品マスタ（箱寸法）

WHERE KAISYACD='<会社コード>' AND SEHNCD='<製品コード>'
```

## 9. セキュリティ注意事項

| 種別 | キーワード | 内容 |
|---|---|---|
| AutoExec | `Workbook_Open` | 起動時に自動実行される初期化処理がある |
| Suspicious | `Open` | ファイル操作系ステートメントを使用 |
| Suspicious | `Call` | 内部マクロ呼び出し（通常の VBA Call） |
| Suspicious | `Chr` | `Chr(13)` を改行として使用（難読化ではない） |
| Suspicious | `Hex Strings` / `Base64 Strings` | バイナリ設定由来と推測される |
| Suspicious | VBA Stomping | P-code と VBA ソースの差異を検知。旧バージョンの Excel で保存した可能性あり |
| 認証情報 | `DSN=ricdb;UID=ric;PWD=t6101` | Oracle DB 接続文字列がコード内にハードコードされている。**パスワードの管理に注意** |
| 認証情報 | `DSN=ricdb;UID=rich;PWD=t6101` | 履歴 DB 用の別接続文字列もハードコードされている |
| 参照エラー | `zaiko -> #REF!` | 名前付き範囲 `zaiko` が壊れた参照を持つ。VBA から直接参照される可能性があり、集計時に予期しないエラーが発生する可能性がある |
