### 2.3 1号機仕掛

**目的**: 1号機照射中仕掛品の在庫金額計算。タイマー値から照射済時間を計算し金額を算出する。

#### カラム構成（行5ヘッダー）

| 列 | ヘッダー | 説明 |
|---|---|---|
| A | 番号 | 連番 |
| C | 受付番号 | — |
| D | 得意先コード | — |
| E | 得意先名 | — |
| F | 製品コード | — |
| G | 製品名 | — |
| H | 線量(kGy) | — |
| I | 装置 | — |
| J | 照射中数 | — |
| K | 照射位置 | — |
| L | 設定時間 | 設定照射時間 |
| M | 開始タイマー | 照射開始時タイマー値 |
| N | 中断時タイマー | 中断時タイマー値 |
| O | 照射状況 | 2=完了、その他=照射中 |
| P | 照射済時間 | `=IF(O2*1=2, N2-M2, IF(M2="","",ROUND(Ric1Timer-M2,2)))` |
| Q | 単位数量 | — |
| R | 単価 | — |
| S | 単位 | — |
| T | 金額 | `=IF(P2="","",ROUND(P2/L2*R2*Q2,0))` |
| U | 備考 | — |
| V | 標準ケース換算 | — |

> **補足**: P2の照射済時間は照射状況が完了（O2=2）の場合 `中断時タイマー - 開始タイマー`、照射中の場合 `月末タイマー(Ric1Timer) - 開始タイマー` で計算。T2の金額は `照射済時間 ÷ 設定時間 × 単価 × 単位数量` の按分計算。

## 6. VBA モジュール仕様

### 6.1 在庫起動.bas

#### `StartUp()`

**処理概要**: ブック起動時に在庫集計済かどうかを確認し、未集計なら入力エリアを活性化、集計済なら集計表作成入力エリアを活性化する。

#### `在庫集計起動()`

**処理概要**: 入力値チェック後にDBからデータを取得し、ファイルを別名保存する。

**処理フロー**:
1. 既集計チェック（`TaihiDay` が空でないとエラー）
2. 必須入力チェック（年・月・氏名・1号機タイマー）
3. `TaihiDay` に現在日時を記録
4. 各在庫・仕掛シートのデータをクリア
5. 各DB抽出処理（在庫抽出/1号機仕掛抽出/23号機仕掛抽出/EB抽出）を実行
6. `月末在庫記録処理` → `D:\月末在庫\yy年mm月末在庫ddhhmmss.xlsm` に別名保存

---

### 6.2 抽出条件設定.bas

#### `在庫抽出()`

**処理概要**: `zaiko`, `sehmst`, `ExSeihinZ` からガンマ照射課在庫データを取得し γ在庫シートに表示する。

```sql
SELECT za.uno, za.kaisyacd, za.kainame, za.sehncd, se.seiname, za.siteisn*1,
       za.syouso*1, za.nyukasu*1, za.misyousu*1, za.syousu*1, za.syukasu*1,
       za.syouzusu - za.syukasu, za.pass*1, za.incnt*1, '',
       e.tanka, TRIM(e.tani), '', e.keijyou, '', e.tanka, TRIM(e.tani), e.keijyou
FROM zaiko za, sehmst se, ExSeihinZ e
WHERE za.kaisyacd = se.kaisyacd AND za.sehncd = se.sehncd
  AND (za.syouzusu - za.syukasu) > 0
  AND e.kaisyacd(+) = za.kaisyacd AND e.sehncd(+) = za.sehncd
ORDER BY za.kaisyacd, za.sehncd
```

### 6.3 集計.bas

#### `集計表作成()`

**処理概要**: γ在庫・EB在庫・1号機仕掛・23号機仕掛の金額列を読み取り、装置別に合計して集計表シートに書き込む。

**処理フロー**:
1. γ在庫シートから装置番号（I列）別に金額（T列）を合計
2. EB在庫シートから金額（P列）を合計
3. 1号機仕掛シートから金額（T列）を合計
4. 23号機仕掛シートから装置番号別に金額（Q列）を合計
5. 集計表シートの `SyuukeiTB` エリアに書き込み

### 6.4 製品単価更新.bas

#### `単価備考更新()`

**処理概要**: γ在庫・EB在庫シートの単価/単位/備考の変更分を `ExSeihinZ` テーブルに記録する。

**処理フロー**:
1. `SyukeiBi`・`SyukaiSya` 入力確認
2. 確認ダイアログ表示
3. γ在庫シートをループ
   - W列（DB単価）≠ R列（現在単価）、X列≠S列、または Y列≠U列 の場合のみ更新
   - `NotTanka` コード、「不要」フラグ付き、受付番号固有は除外
   - `ExSeihinZ` テーブルへINSERT/UPDATE
4. EB在庫シートも同様に処理

## 9. セキュリティ注意事項

| 種別 | キーワード | 内容 |
|---|---|---|
| AutoExec | `Workbook_Open` | ブック起動時に自動実行。DB接続・初期化が行われる |
| AutoExec | `Workbook_BeforeClose` | 終了時に自動実行 |
| Suspicious | `Open` | ファイルオープン操作が含まれる可能性 |
| Suspicious | `Call` | DLL呼び出しの可能性 |
| Suspicious | `MkDir` | ディレクトリ作成操作（`D:\月末在庫` の作成は正常用途） |
| Suspicious | `ActiveWorkbook.SaveAs` | 別名保存操作（月末在庫ファイルの保存は正常用途） |
| Suspicious | `Chr` | 文字列難読化の可能性（`Chr(13)` = 改行は正常用途） |
| Suspicious | Hex Strings | 16進数エンコード文字列が検出された |
| Suspicious | Base64 Strings | Base64エンコード文字列が検出された |
| **Critical** | **VBA Stomping** | **VBAソースコードとP-codeが異なる（VBA Stomping検出）。悪意あるコードの隠蔽に使用された可能性がある。** |

> **重大な注意**:
> 1. **VBA Stomping が検出された**。これは実際に実行されるP-codeとソースコードが異なることを意味し、セキュリティ上の重大なリスクがある。専門家によるマルウェア解析を推奨する。
> 2. DB接続パスワードが平文で記載されている: `"DSN=ricdb;UID=ric;PWD=t6101"`
> 3. 名前付き範囲 `Zaiko`、`SikaR1` に `#REF!` エラーがある。
